# トラブル対応・事故パターン集

## 緊急停止

何かおかしいと思ったら即座に全停止:

```bash
python main.py --agent kill-switch-on
```

再開するとき:
```bash
python main.py --agent kill-switch-off
```

## 事故パターンと対策

### 事故1: アカウント凍結（実際にやらかした）

**症状**: 投稿できなくなる・ログインできなくなる

**原因**: 1分で10投稿など、人間には不可能な速度でbotと判定された

**対策（実装済み）**:
- 最低投稿間隔: 1時間
- 1日上限: 10投稿
- タイムスロット分散（朝8時〜深夜1時）

**絶対にやってはいけないこと**:
- `MIN_POST_INTERVAL_HOURS` を0にする
- `MAX_DAILY_POSTS` を極端に増やす
- `config.py` の投稿制限を外す

---

### 事故2: 同じ投稿が30回ループ（実際にやらかした）

**症状**: アカウントの投稿一覧がほぼ同じ内容で埋まる

**原因**: AIは最適解（一番バズりやすい型）に収束し続ける。類似度チェックがなかった。

**対策（実装済み）**:
- 過去100件との類似度 0.85以上 → 自動棄却
- 直近3件と同パターン → 使用禁止
- 15種のパターンを強制ローテーション
- 同テーマ3連続 → 自動切り替え

---

### 事故3: 投稿が止まる

**確認手順**:
```bash
# 1. KILL_SWITCHが発動していないか
python main.py --agent supervisor-status

# 2. キューに投稿があるか
python -c "
import json
from pathlib import Path
q = json.loads(Path('data/queue.json').read_text(encoding='utf-8'))
pending = [p for p in q if p.get('status') == 'queued']
print(f'キュー残: {len(pending)}件')
"

# 3. APIキーが生きているか
python main.py --status
```

**よくある原因**:
- `data/KILL_SWITCH` ファイルが残っている → `kill-switch-off` で解除
- Threadsトークンが期限切れ（60日） → Meta Developersでトークン再発行
- キューが空 → `python main.py --agent writer --mode review` で投稿生成

---

### 事故4: メトリクスが取れない

**原因**: Threads APIの `insights` エンドポイントには `threads_manage_insights` 権限が必要

**対策**:
1. Meta for Developers → アプリ → Threads API
2. スコープに `threads_manage_insights` が含まれているか確認
3. トークンを再発行（スコープ追加後）

---

### 事故5: Writerが投稿を全部棄却する

**症状**: `Added 0 posts to queue/drafts` が続く

**原因と対策**:

| 原因 | 確認方法 | 対策 |
|---|---|---|
| 類似度が高すぎる | ログに `Similarity 1.000` が並ぶ | `data/queue.json` と `research_cache.json` をリセット |
| APIキーなし・モックが重複 | `python main.py --status` | Anthropic APIキーを設定する |
| research_cacheが空 | `data/research_cache.json` を確認 | `python main.py --agent researcher` を先に実行 |

---

### 事故6: Threadsトークン期限切れ（60日後）

**症状**: 投稿APIが401エラーを返す

**対策**:
1. Meta for Developers → 「APIを調査する」 → トークン再発行
2. `.env` の `THREADS_ACCESS_TOKEN` を更新

**長期トークンへの変換（根本対策）**:
```
https://graph.threads.net/refresh_access_token
  ?grant_type=th_refresh_token
  &access_token={現在のトークン}
```
→ 長期トークンに変換すると60日ごとに自動更新される

## ログの見方

```bash
# リアルタイムでログを見る（Windows）
Get-Content data\run.log -Wait -Tail 20

# 最新20行だけ確認
python -c "
from pathlib import Path
lines = Path('data/run.log').read_text(encoding='utf-8').splitlines()
print('\n'.join(lines[-20:]))
"
```

**ログレベルの意味**:
- `[INFO]` : 正常動作
- `[WARNING]` : 注意（棄却・スキップなど）
- `[ERROR]` : エラー（API失敗など）
- `[CRITICAL]` : 緊急停止が発動
