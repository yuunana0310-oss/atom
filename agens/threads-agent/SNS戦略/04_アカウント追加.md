# アカウント追加・ジャンル展開手順

## 設計思想

**コードは一切触らない。ナレッジファイルを差し替えるだけ。**

`knowledge/` フォルダの4ファイルがアカウントの「人格」。
これを書き換えるだけで転職でも美容でもAIでも同じシステムが動く。

---

## 手順

### 1. knowledge フォルダをコピー

```bash
cd C:\Users\yuuna\agens\threads-agent
xcopy knowledge knowledge_転職 /E /I    # 現在のをバックアップ
xcopy knowledge knowledge_AI /E /I      # 新ジャンル用を作成
```

### 2. 以下の4ファイルを書き換える

```
knowledge/
├── account_profile.json   ← アカウントの人格設定
├── theme_tree.json        ← ジャンルのテーマ構造
├── ng_words.json          ← 使ってはいけないワード
└── post_patterns.json     ← 15種の投稿パターン（通常は変更不要）
```

### 3. account_profile.json の書き方

```json
{
  "account_name": "AIアカウント",
  "target_persona": "AIに興味があるが何から始めればいいか分からない20〜35歳の会社員",
  "tone": "親しみやすく・具体的・背中を押す",
  "niche": "AI活用",
  "affiliate_category": "オンラインスクール・AIツール",
  "ng_words": ["絶対稼げる", "楽して", "誰でも"],
  "posting_style": "体験談型 + 短文完結"
}
```

### 4. theme_tree.json の書き方

```json
{
  "プロンプト": ["ChatGPTプロンプト術", "画像生成プロンプト", "仕事効率化"],
  "ツール比較": ["ChatGPT vs Claude", "画像生成AI比較", "コーディングAI"],
  "活用事例": ["副業での使い方", "仕事術", "日常生活"],
  "稼ぎ方": ["AIライティング", "AI画像販売", "AIコンサル"]
}
```

### 5. ng_words.json の書き方

```json
{
  "ng_words": [
    "絶対稼げる",
    "楽して",
    "誰でも",
    "今すぐ",
    "損しない"
  ]
}
```

### 6. dataフォルダをリセット

新アカウントは過去データなしでスタートするためリセットする。

```bash
# 既存のdataをバックアップ
xcopy data data_転職バックアップ /E /I

# dataを空にする（初期化）
python -c "
import json
from pathlib import Path
for f in ['post_history.json', 'queue.json', 'drafts.json',
          'research_cache.json', 'feedback_instructions.json',
          'supervisor_state.json']:
    Path(f'data/{f}').write_text('[]', encoding='utf-8')
print('データリセット完了')
"
```

### 7. 切り替え完了・起動

```bash
python main.py --agent all --mode review
```

## スケール試算

| アカウント数 | 月収目安（完全自動） | 月収目安（5%人間） |
|---|---|---|
| 1 | 〜5万円 | 〜40万円 |
| 3 | 〜15万円 | - |
| 5 | 〜25万円 | - |
| 10 | 〜50万円 | - |

**5%人間は1アカウントに集中するから複数展開しにくい。**
複数展開するなら完全自動で量を取る戦略が現実的。
