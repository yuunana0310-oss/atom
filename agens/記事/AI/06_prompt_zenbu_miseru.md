## 1. なぜ提案書にChatGPTが効くのか

提案書が通らない原因の多くは、「構成」と「言葉選び」にある。

内容は正しいのに、相手に伝わらない順番で書いてしまっていたり、「なんとなく弱い言葉」で終わらせてしまっていたりする。

ChatGPTが得意なのは、まさにそこだ。
「この流れで伝えるとどう聞こえますか？」と聞くだけで、客観的なフィードバックが返ってくる。

自分の頭の中を整理するための壁打ち相手として使う——それがいちばん効果的な使い方だと思っている。

---

## 2. 通った提案書の背景と、評価された理由

先月通ったのは、中小企業向けSNS運用代行サービスの提案書だった。

- **クライアント**: EC系の中小企業（社員10名以下）
- **競合状況**: すでに他の業者にも見積もりを依頼していた
- **条件**: 月5万円の予算は固定——価格では差がつけられない

正直、厳しい条件だった。
知名度でも価格でも勝てない。だから「提案書の中身」で差をつけるしかなかった。

採択後、クライアントから「他の業者の提案書と比べて、うちの状況をちゃんと理解してくれてる感じがした」と言われた。
**相手の課題をどれだけ具体的に言語化できるか**——それが採択の決め手だったと今でも思っている。

## 3. やり取りの4フェーズ

提案書を作るとき、ChatGPTとのやり取りは毎回だいたい4つのフェーズに分かれる。

**フェーズ1：提案書の構成を考える**
まず「何をどの順番で伝えるか」の骨格を作る。
ここを曖昧にしたまま書き始めると、後から全部書き直しになる。

**フェーズ2：各セクションを書かせる**
骨格が固まったら、1セクションずつ書いてもらう。
一気に全部書かせようとしない。確認しながら進めることで、ズレが小さいうちに直せる。

**フェーズ3：修正・ブラッシュアップ**
全体を通して読んで、「ここが弱い」「ここが長い」と感じた箇所を修正する。
このフェーズで提案書の印象がかなり変わる。

**フェーズ4：クライアント目線の最終確認**
「クライアントの立場で読んだとき、刺さるか？」を ChatGPT に確認してもらう。
自分では気づかない「当事者以外には伝わらない前提」を指摘してくれることが多い。

### プロンプト① 提案書の構成を考える

> **効果**: 「何を書けばいいかわからない」状態から、5分で骨格ができた

```
あなたはBtoBの提案書作成の専門家です。
以下の条件で、提案書の構成案を作ってください。

【案件情報】
- 提案内容：SNS運用代行（月5万円）
- クライアント：EC系の中小企業（社員10名以下）
- クライアントの課題：SNSを始めたいが社内に担当者がいない
- 競合状況：他社も見積もりを取っている
- 差別化ポイント：業界特化の実績と、毎週のレポート提供

【要件】
- 読み手は経営者（専門用語は避ける）
- A4で2〜3ページに収める
- 「なぜうちに頼むべきか」が伝わる構成にする

まず構成案（見出しレベル）を5〜7項目で提示してください。
```

### プロンプト② 冒頭の「課題提示セクション」を書かせる

> **効果**: 「御社の現状では〜」という書き出しで、読み手がすぐ「うちのことだ」と感じる文章になった

```
提案書の冒頭セクション「現状の課題と機会」を書いてください。

【前提情報】
- クライアントはSNSに興味があるが、担当者がおらず後回しにしている
- 競合他社はすでにSNSで成果を出している
- このまま何もしなければ機会損失が続く

【制約】
- 200字以内
- 「御社の現状では〜」という形で始める
- 恐怖を煽らず、「機会がある」というポジティブな書き方にする
- 経営者にすぐ伝わるシンプルな日本語で
```

### プロンプト③ 「なぜうちに頼むべきか」のセクションを書かせる

> **効果**: 「〜できます」ではなく「〜してきました」という実績ベースの語尾に変えたことで、説得力が一段上がった

```
提案書の「選ばれる理由」セクションを書いてください。

【アピールしたい強み】
1. 同業種（EC・小売業）での支援実績が10社以上ある
2. 毎週レポートを提出し、数字で進捗を共有する
3. 担当者1名が一貫して対応（窓口が変わらない）

【制約】
- 各強みを2〜3文で説明する
- 「〜ができます」ではなく「〜してきました」「〜しています」という実績ベースの語尾にする
- 全体で300字以内
```

---

### プロンプト④ 価格・プラン説明のセクションを書かせる

> **効果**: 「1本あたり約4,200円」という換算を入れることで、読み手の「高いかも」という感覚が薄れた

```
「料金プラン」セクションを書いてください。

【プラン情報】
- 月額5万円（税別）
- 含まれるもの：投稿作成（月12本）、コメント対応、週次レポート
- 初月はトライアルとして2万円引き

【制約】
- 「高い」と感じさせない書き方をする
- 1本あたりの単価に換算した表現を入れる（月12本で割ると1本あたり約4,200円）
- 最後に「まずはトライアルから試せます」という流れで締める
```

---

### プロンプト⑤ 全体を通して「弱い部分」を指摘させる

> **効果**: 自分では気づかなかった「読み手が置いてきぼりになる箇所」を3点ピンポイントで指摘してくれた

```
以下の提案書の全文を読んで、改善すべき点を指摘してください。

【提案書全文】
（ここに書いた提案書を貼り付ける）

【指摘してほしい観点】
1. 読み手（経営者）が「で、結局何をしてくれるの？」と思いそうな箇所
2. 言葉が抽象的すぎて具体性が薄い箇所
3. 流れが不自然で読みにくい箇所

優先度の高い順に3点、具体的な修正案も一緒に教えてください。
```

---

### プロンプト⑥ 修正後の文章をブラッシュアップさせる

> **効果**: 自分で修正した文章をさらに一段磨く用途で使った。「言い換え案を3パターン出して」と追加するとさらにいい

```
以下の文章をより説得力のある表現に書き換えてください。

【元の文章】
（修正したい部分を貼り付ける）

【ブラッシュアップの方針】
- 経営者が読んで「それ、うちのことだ」と感じられる具体性を加える
- 「〜と思います」「〜と考えています」という弱い語尾を減らす
- 全体のトーンは変えず、言葉の力だけを強くする

元の文章と同じ構成で、書き換え後のバージョンを出してください。
```

---

### プロンプト⑦ クライアント目線の最終チェック

> **効果**: 「ここ、専門用語すぎて経営者には伝わらないかも」という指摘をもらい、提出前に修正できた

```
あなたは今からEC系中小企業の経営者になってください。
以下の提案書を読んで、正直な感想を教えてください。

【提案書全文】
（最終版を貼り付ける）

【聞きたいこと】
1. 読んでみて「これ、いいかも」と感じた点はどこか
2. 「ここが気になる」「もう少し説明してほしい」と思った点はどこか
3. この提案書を受け取ったとして、発注したいと思えるか（理由も教える）

忖度なしで答えてください。
```

## 5. 続けて使うためのコツ

プロンプトをただコピペするより、**【案件情報】の部分だけ書き換えて使い回す**のがいちばん効率がいい。

骨格は同じで、案件の中身だけ差し替えるイメージ。
5回も使えば、自分なりのアレンジが自然にできてくる。

もう一つ意識してほしいのは、**ChatGPTを「最終稿を書いてくれる人」ではなく「壁打ち相手」として使う**こと。
全部任せようとすると、どこかふわっとした文章になる。
「方向性を決める→書かせる→自分で手直しする」のサイクルが、精度をいちばん上げる。
