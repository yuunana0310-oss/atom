## 1. ゼロから書き始めない——「型」を借りるという発想

多くの人がやってしまうのが、「テーマを決めたらいきなり書き始める」こと。
これが一番きつい。

note上位記事には、読まれる理由がある。**冒頭で共感を取る、課題を設定する、解決策を順番に出す**——そういう「型」が存在する。
その型を自分で発明しようとするから疲弊する。

だから最初にやるべきことは、「参考記事の構造を解読する」です。

AIはその解読作業を、数分で終わらせてくれます。

---

## 2. AIを使う前の準備（参考記事を3本集める）

AIに書かせる前に、まず参考記事を集めます。

**集め方のポイント**

- noteの検索で、自分が書きたいテーマに近い記事を探す
- いいねが50以上ついているものを選ぶ
- 有料記事でなくていい（無料記事で十分）
- 3本あれば十分

この3本が「AIへの教師データ」になります。
参考記事なしでAIに書かせると、どこかで見たような当たり障りのない文章が出てくる。それではnoteでは埋もれます。

## 3. 全体ワークフローの概要

自分が使っているワークフローはシンプルです。

```
参考記事3本 →【Step1】構造分析 →【Step2】自分のテーマで再構築
→【Step3】タイトル案を複数出す →【Step4】採点・改善ループ
→ 85点以上で完成
```

各ステップに使うプロンプトは、有料部分にそのまま載せています。

## 4. 【Step1】参考記事の構造分析プロンプト

参考記事をAIに読ませて、構造を解読させます。
記事本文をコピーして、以下のプロンプトの末尾に貼り付けるだけ。

```
以下のnote記事（本文貼付）の構造を分析して:
・冒頭フック（読者の悩みをどう代弁しているか）
・読者の課題設定の仕方
・解決策の提示順序
・締めとCTAの作り方
・タイトルの言葉選びのパターン

---
【本文貼付箇所】
```

3本分やると、「このジャンルでウケる型」が見えてきます。
「どの記事も最初の2行で悩みを代弁している」「解決策は必ず3〜5個で番号付き」——そういうパターンが揃ってきたら次のステップへ。

---

## 5. 【Step2〜4】記事生成・タイトル案・採点ループのプロンプト

### Step2: 自分のテーマで再構築

```
同じ構造で、テーマを「〇〇」に変えて記事を書いて。
読者は「△△で悩んでいる30〜40代の会社員」。
口調はカジュアルだが知的。煽り表現は使わない。
文字数は1,500〜2,000字。
```

`〇〇` と `△△` だけ自分の記事に合わせて変える。
あとはAIが「さっき分析した型」に従って書いてくれます。

---

### Step3: タイトル案を複数生成

```
この記事に合うタイトルを10個出して。
以下の条件を守ること:
・【 】ブラケットを使うパターン
・数字を入れるパターン
・「〇〇した話」パターン
・専門家権威を示すパターン
```

10個出すことで「これとこれを組み合わせたら良さそう」という発想が生まれます。
自分は大体2〜3案を混ぜて最終タイトルを作ります。

---

### Step4: 採点・改善ループ

```
この記事を以下の観点で10点満点で採点して:
・フックの強さ
・読者への共感度
・具体性・再現性
・読みやすさ
・CTAの自然さ
改善点を3つ教えて
```

合計50点満点のうち42点（=84%）、つまり**85点以上になるまでこのループを繰り返します**。

最初は50〜60点台が出ることも多い。そこで「改善点を3つ教えて」の指示を受けて修正し、また採点させる。
2〜3回繰り返すと85点以上に乗ってくることが多いです。

## 6. 実際に使ってみた結果

このワークフローを使い始めたのは、AIカテゴリと医療系の記事で試したのがきっかけです。

**AIカテゴリ**では、参考記事に「Claude Codeを非エンジニアが使った体験談」を使って、似た構造で「PT×AI開発」の記事を書きました。
公開後48時間でいいねが15件。以前の自分の記事がだいたい0〜2件だったので、手応えを感じました。

**医療系カテゴリ**では、「専門家権威×季節フック」の型を使って腰痛ストレッチの記事を書いたら、いいねが23件。
型を意識するだけで、こんなに反応が変わるのかと少し驚きました。

---

**ワークフロー全体をまとめると**

1. 参考記事3本を集める
2. Step1プロンプトで構造分析
3. Step2プロンプトで記事生成
4. Step3プロンプトでタイトル案を出す
5. Step4プロンプトで採点→85点以上まで改善
6. 公開

慣れると1記事あたり1〜2時間で仕上がります。
AIを使う前は半日かかっていたことを考えると、体感としてはかなり楽になりました。

## まとめ

AIで記事を書くといっても、「プロンプトを貼れば勝手に完成する」ではありません。
**参考記事の構造を読む目を養い、型を意識してAIに指示する**——この準備があって初めて、AIが力を発揮します。

まずは手元に参考記事を3本集めることから始めてみてください。
