# 論文投入・検証ワークフロー

## 目的

医学論文をシステマティックに RAS（Research Automation System）に投入し、医療・リハビリ関連のコンテンツ作成時に高い信頼性で参照可能にする。

## ワークフロー全体

```
[論文テキスト] 
    ↓
[/summarize-papers-to-ras で PT向けサマリー化]
    ↓
[RAS_IMPORT_BOX に保存]
    ↓
[import_research.py で Vault に移行・インデックス化]
    ↓
[SynthesisRAG で RAG 検索対応に]
    ↓
[コンテンツ作成時に RAG で参照]
```

---

## フェーズ別実行手順

### フェーズ 1: 論文テキストの用意

**入力ソース:**
- PubMed のテキスト抽出
- Google Scholar から PDF をテキスト化
- 査読済みジャーナルの HTML から抽出

**確認事項:**
- [ ] 論文タイトルが含まれているか
- [ ] Abstract が含まれているか
- [ ] Methods / Results セクションが含まれているか
- [ ] 出版年が明記されているか

### フェーズ 2: 論文の存在確認（品質管理）

**実行方法:** WebSearch × PubMed で論文の実在性を確認

**確認項目:**
- [ ] 論文がPubMedに登録されているか
- [ ] DOI（Digital Object Identifier）が正確か
- [ ] 著者名と出版年が一致しているか
- [ ] ジャーナルの査読レベルは十分か

**信頼度判定:**
- **A+（完全確認）**: DOI確認、著者名確認、結果データが一致 → 臨床判断の根拠に可
- **A（ほぼ確認）**: PubMedで登録確認、出版年が一致 → コンテンツ作成に使用可
- **B（部分確認）**: 論文タイトルのヒットはあるが、著者名が部分一致 → 追加確認推奨
- **C（未確認）**: テキスト検索で確認できず → 使用不可

### フェーズ 3: PT向けサマリー化

**実行方法:**
```bash
/summarize-papers-to-ras
# または
python C:\Users\yuuna\agens\knowledge\papers_to_vault.py
```

**生成内容の確認:**
- [ ] PT が臨床判断に使用できる情報か
- [ ] 数値データが原論文と一致しているか
- [ ] 禁忌・安全性が記載されているか
- [ ] エビデンスレベルが明記されているか

### フェーズ 4: RAS_IMPORT_BOX から Vault への移行

**自動実行:**
```bash
python C:\Users\yuuna\agens\knowledge\import_research.py
```

**処理内容:**
- RAS_IMPORT_BOX のファイルを読み込み
- H1見出しで自動分割
- フロントマター付与
- `research_synthesis/vault/` に移動
- SynthesisRAG インデックス更新

### フェーズ 5: RAG 検索対応状態の確認

```python
from research_synthesis.synthesis_rag_engine import SynthesisRAG

rag = SynthesisRAG()
results = rag.search("股関節OAの運動療法")

for result in results:
    print(f"タイトル: {result['title']}")
    print(f"精度: {result['score']}")
```

**確認項目:**
- [ ] 関連キーワード検索で該当論文が上位に表示されるか
- [ ] スコア（精度）が 0.7 以上か
- [ ] 出典が正確に記録されているか

## PT向けサマリー標準フォーマット

### 背景（2-3文）
臨床的な必要性と研究背景

### 対象（箇条書き）
- 対象患者: [年齢、性別、診断名]
- サンプルサイズ: n = 数値
- 選定基準: [基準詳細]

### 方法（箇条書き）
- 介入内容
- 実施期間
- 測定項目

### 結果（数値データ）
主要評価項目の改善値・p値を記載

### 結論
- エビデンスレベル: I / II / III / IV / V
- 信頼度: 高 / 中 / 低
- 制限事項

### 臨床応用（最重要）
- **推奨患者**: [具体例]
- **実装方法**: [PT が実装可能な具体的内容]
- **注意点**: [禁忌・安全性・モニタリング]

---

## RAS 投入時の確認チェック

- [ ] 論文存在確認は完了（品質 A 以上）
- [ ] タイトルが正確か
- [ ] 背景で「なぜ必要か」が明記されているか
- [ ] 対象患者が PT に分かりやすいか
- [ ] 方法が「再現可能」か
- [ ] 結果が数値を含んでいるか
- [ ] エビデンスレベルと信頼度が明記されているか
- [ ] 臨床応用が「今日から実装できる」か
- [ ] 禁忌・安全性が記載されているか

---

## 推奨実行頻度

- **単発**: 1-2論文 → 即実行
- **定期**: 複数論文継続投入 → 週1回のバッチ処理
- **大規模**: 50論文以上 → スケジュール機能活用
