# エージェント設計

## 全体フロー

```
[Researcher]  YouTubeからネタ収集
     ↓
[Analyst]     過去データ分析 → Writerへの指示書生成
     ↓
[Writer]      投稿文生成・自動採点（7.0以上のみ通過）
     ↓
  autoモード → queue.json → [Poster] → Threads投稿
  reviewモード → drafts.json → 人間10分チェック → queue.json → [Poster] → Threads投稿
     ↓
[Fetcher]     1h / 6h / 24h でメトリクス取得
     ↓
[Analyst]     次のサイクルへフィードバック（自律改善）
```

**設計原則: 1エージェント = 1タスク**
複数の責務を1つに混ぜると精度が落ちる。

---

## 各エージェントの詳細

### Researcher（ネタ収集）
- `knowledge/theme_tree.json` のテーマツリーを見て、ネタが足りないテーマを自動判断
- YouTubeを検索し動画の概要・タイトルからネタを抽出
- `data/research_cache.json` に保存
- APIキーなし → 組み込みモックデータ15件で動作

### Analyst（分析）
- `data/post_history.json` の投稿メトリクスを分析
- 「この投稿パターンが伸びてる」「このテーマは飽きられてる」を判定
- `data/feedback_instructions.json` に指示書を書き出す
- WriterはこれをもとにAIが次のバッチで方針を変える（AIがAIにフィードバック）

### Writer（投稿生成）
- 参照するもの: research_cache + feedback_instructions + account_profile + post_patterns
- **自動採点 10項目 × 10点**:
  1. フックの強さ
  2. 有益性
  3. 具体性
  4. テンポ感
  5. ペルソナ一致度
  6. オリジナリティ
  7. 読みやすさ
  8. 感情的共鳴
  9. 行動喚起
  10. Threads適性
- **平均7.0未満 → 書き直し（最大2回）→ 棄却**
- **過去100件と類似度 0.85以上 → 棄却**
- **直近3件と同じパターン → 使用禁止（15種ローテーション）**

### Poster（投稿実行）
- `data/queue.json` のstatus=queuedの投稿を時間スロットで投稿
- 投稿スロット（1日最大10本）: 8:00 / 10:00 / 12:00 / 14:00 / 16:00 / 18:00 / 20:00 / 21:30 / 23:00 / 1:00
- 最低間隔1時間（bot判定回避）
- `comment_hook`型 → コメント欄に続きを自動投稿
- アフィリエイトあり → コメント欄にPRリンクを自動配置

### Fetcher（メトリクス取得）
- **3段階チェックポイント**:
  - 1h後（0.5〜2h）: 初速チェック・バズの兆候確認
  - 6h後（5〜8h）: 中間計測
  - 24h後（20〜28h）: 最終計測
- 取得データ: 閲覧数 / いいね / リプライ / リポスト / 引用
- `post_history.json` に `metrics_1h` / `metrics_6h` / `metrics_24h` で保存

### Supervisor（監視）
- 全エージェントをラップして実行（エラーを検知）
- **連続3回エラー → KILL_SWITCH自動生成 → 全停止**
- `data/KILL_SWITCH` ファイルが存在する限り全エージェントが動かない
- `data/supervisor_state.json` にエラーログを記録

## 投稿パターン15種

| ID | 名称 | 概要 |
|---|---|---|
| short_complete | 短文完結 | 3〜5行で完結 |
| comment_hook | コメント誘導型 | 本文で問い、コメント欄に続き |
| thread_expand | ツリー展開型 | 返信で内容を展開 |
| list_format | リスト系 | 番号付きリスト |
| expose_type | 暴露系 | 業界の裏側を暴露 |
| demand_check | 需要確認型 | 「〇〇な人いる？」形式 |
| experience_share | 体験談型 | 一人称で体験を語る |
| question_answer | Q&A型 | よくある質問に答える |
| myth_bust | 誤解解消型 | 一般的な誤解を正す |
| comparison | 比較型 | AとBを比較 |
| timeline | タイムライン型 | 時系列で整理 |
| checklist | チェックリスト型 | 確認項目をリスト化 |
| warning | 注意喚起型 | やってはいけないことを警告 |
| tip_share | ノウハウ共有型 | 具体的なtipsを共有 |
| story_arc | ストーリー型 | 起承転結で語る |

## データファイル一覧

| ファイル | 用途 | 消すと壊れるか |
|---|---|---|
| `data/research_cache.json` | Researcherのネタストック | △（再収集すればOK） |
| `data/queue.json` | 投稿待ちキュー | ✓（投稿予定が消える） |
| `data/drafts.json` | 5%人間モードの下書き | △ |
| `data/post_history.json` | 投稿済み履歴+メトリクス | ✓（分析・重複チェックが壊れる） |
| `data/feedback_instructions.json` | AnalystからWriterへの指示書 | △ |
| `data/supervisor_state.json` | エラーカウント | △ |
| `data/KILL_SWITCH` | 緊急停止フラグ | -（このファイルがあると全停止） |
