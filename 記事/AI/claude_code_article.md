## 1. Claude Codeとは何か、使う前の誤解

Claude Codeは、Anthropicが開発したAIを使った作業支援ツールだ。

名前に「Code」とついているせいで、「プログラマー専用」と思い込んでいた。実際に触れてみると、それは半分だけ正しかった。

ターミナル（黒い画面）上でAIに指示を出すと、それを実行してくれる。コードを書く人なら、コードを書いてくれる。でも、コードを書かない人でも、ファイル操作・文章整理・情報まとめなど、「繰り返し作業」に幅広く使える。

要するに、**「指示すれば動いてくれる、PCの中にいるアシスタント」**に近い。

ブラウザ版のChatGPTやClaudeと何が違うか。一言でいえば、**「自分のPCのファイルに直接触れる」**ことだ。ここが、使ってみて一番驚いた点だった。

---

## 2. セットアップで詰まった話（正直に）

インストールは、手順通りに進めれば30〜40分で終わる。

途中、英語のエラーが1回出た。私はそのエラー文をコピーして、Claudeのブラウザ版に貼り付け「これはどういう意味？　どう直す？」と聞いた。

返ってきた回答をターミナルに打ち込んだら、解決した。

「ターミナルが怖い」という感覚は、最初の30分だけだった。その壁を超えると、あとはほぼチャット感覚で操作できる。

怖かったのは本当だ。でも、怖かったのはそれだけで、難しかったわけではなかった。

---

## 3. 3週間の変化：週ごとの記録

### 1週目：とにかく迷った

最初の1週間は、「何をやらせるか」が全然わからなかった。

試しに「このテーマで記事の構成案を出して」と頼んでみた。出てきた構成は悪くなかった。でも、「これ、ChatGPTと何が違うんだろう」という感覚が正直あった。

**この1週間は、使えた気がしなかった。**

---

### 2週目：「PCの中で動く」ことに気づいた

2週目に入って、ようやく気づいた。

Claude Codeは、自分のPCのフォルダやファイルに直接アクセスできる。「Downloadsフォルダの中の請求書を、クライアント別のフォルダに振り分けて」と指示してみた。

**30秒で終わった。毎月40分かけていた作業が。**

この瞬間、初めて「あ、これは使えるかもしれない」と思った。

---

### 3週目：仕事のリズムが変わった

3週目になると、自然と「これ、Claude Codeに任せよう」と思える場面が出てくるようになった。

具体的に変わった作業はこの3つだ。

**① リサーチの下準備が速くなった**
記事1本の情報収集に、以前は1〜2時間かかっていた。「このテーマの論点を調べて、箇条書きで出して」と指示するだけで、10〜15分で叩き台が出てくる。そのまま使うわけではないが、「ゼロから調べる」が「修正・選別する」に変わった。

**② 繰り返しの文字処理が消えた**
クライアントから届く資料を、特定のフォーマットに変換する作業があった。毎回コピペで30分かかっていたものが、5分以内に終わるようになった。

**③ 指示の言語化力が上がった**
これが一番予想外だった。Claude Codeに曖昧な指示を出すと、的外れな結果が返ってくる。だから自然と、「何を、どこまで、どんな形式で」を明確にする習慣がついた。これが、クライアントへのメールや提案書の書き方にも波及した。取引先から「最近わかりやすくなった」と言われたのは、少し嬉しかった。

---

## 4. 変わらなかったこと・期待外れだったこと

正直に書く。

### 収入はすぐには変わらない

3週間でClaude Codeを使いこなせるようになっても、月収はすぐに上がらない。

浮いた時間を何に使うか、それは自分で決めなければならない。ツールが勝手に稼いでくれるわけではない。

### 「何をやらせるか」を考えるのが思ったより疲れる

使いこなせるようになるほど、「もっとうまく使えるはず」という気持ちが出てくる。試行錯誤の時間は、思っていたより多くかかった。

「便利なツールを手に入れた」というより、「付き合い方を学ぶ相手が増えた」という感覚に近い。

## 5. 実際に使ったプロンプト・指示例 5つ

ここが、この記事で一番伝えたかった部分だ。

「何を頼めばいいかわからない」という人のために、私が実際に使った指示をそのまま公開する。

---

**① フォルダ整理（2週目に使った最初の指示）**

```
Downloadsフォルダの中にある請求書のPDFを、
ファイル名に含まれるクライアント名ごとにフォルダを作って振り分けてください。
クライアント名は「〇〇株式会社」という形式で含まれています。
```

→ 30秒で終わった。毎月40分の作業が消えた。

---

**② 記事リサーチの下準備**

```
「フリーランスライターが単価を上げる方法」というテーマで記事を書きます。
読者は月収10〜15万円で伸び悩んでいるライターです。
論点になりそうな切り口を10個、箇条書きで出してください。
各項目に「なぜこれが読者に刺さるか」を一言添えてください。
```

→ 記事の構成を考える時間が半分になった。

---

**③ 議事録の整形**

```
添付のテキストファイルはZoomの自動文字起こしです。
話者ごとに発言を整理して、決定事項と次のアクションを最後にまとめてください。
```

→ 30分かかっていた議事録整理が10分以下に。

---

**④ クライアントへのメール文作成**

```
以下の状況でクライアントへのお詫びメールを書いてください。
・納品が2日遅れた
・理由：取材先の回答が遅れた（先方都合）
・次回の納品スケジュールは守れる見込み
丁寧すぎず、誠実に。300字以内で。
```

→ 「こんなに早く書けるのか」と思った。感情を入れすぎない塩梅が自然だった。

---

**⑤ 複数ファイルの情報まとめ**

```
このフォルダの中にある5本の記事を読んで、
各記事が「どんな読者に」「何を解決するために書かれているか」を
一覧表（記事名・ターゲット・解決する悩み）にまとめてください。
```

→ ポートフォリオ整理に使った。自分の書いた記事の傾向が一目でわかった。

---

## 6. 月収10万円台の人間が使うべきか？　私の結論

使った方がいい。ただし、**期待値を整えてから使った方がいい。**

Claude Codeは魔法ではない。繰り返し作業を減らし、思考の補助をしてくれるツールだ。

でも、「何を任せるか」を考える力が自分にあれば、確実に仕事のやり方は変わる。

3週間で変わったのは、スピードよりも「仕事の解像度」だったかもしれない。気がつけば、「この作業、もっとうまくできるはず」という視点が、常に頭の片隅にあるようになっていた。

**それは、月収を上げるよりも先に手に入れておくべき感覚だと、今は思っている。**

## まとめ

非エンジニアでも、Claude Codeは使える。怖いのは最初の30分だけだ。

変わったこと：繰り返し作業・リサーチ時間・指示する力。

変わらなかったこと：すぐに収入が増えること。

でも、使い続けるほど「仕事の解像度」が上がっていく。それだけは自信を持って言える。
