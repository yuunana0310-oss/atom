# 第1章：医療職こそ、今すぐClaude Codeを使うべき理由

## 医療職の「時間貧乏」問題

厚生労働省の調査によると、看護師の時間外労働の主な原因は「記録・書類業務」です。理学療法士・作業療法士・介護士も同様です。

臨床の判断、患者さんとのコミュニケーション、家族対応——それが医療職の本来の仕事のはずなのに、気づけば1日の業務の3割近くが「文章を作る仕事」に使われています。

これは個人の努力や工夫で解決できる問題ではありません。**構造的に変えるしかない。**

その構造を変えるツールが、Claude Codeです。

---

## AIの波は、医療現場にも来ている

「AIって、エンジニアとか経営者が使うものでしょ？」

そう思っていませんか。私も1年前はそうでした。

でも今、看護師がAIで看護記録の下書きを作り、理学療法士がAIで患者説明の資料をつくり、医師がAIで紹介状の文面を整える——そんな現場が少しずつ増えています。

2025年以降、医療現場へのAI導入は急速に進んでいます。早く使いこなした人が、**「AIを使える医療職」として圧倒的に有利になる時代**が来ています。

医療職は「専門知識がある」という強みを持っています。AIはその専門知識を**何倍にも増幅させるツール**です。使わない理由はない。

## 「ChatGPTは試した。でも続かなかった」という方へ

AIを試したことがある方に聞くと、こんな声が多いです。

> 「ChatGPTを使ってみたけど、医療の記録には使いにくかった」
> 「何を入力すれば良いかわからなくて、結局やめた」
> 「出てきた文章が硬くて、そのまま使えなかった」

これは、あなたの使い方が悪かったのではありません。**ツールの選び方と指示の出し方の問題**です。

ChatGPTとClaude Codeは、同じ「AI」でも設計の思想がまったく違います。医療職の業務との相性も大きく異なります。

---

## 「ChatGPTで十分じゃないの？」という疑問に答える

AIというと、まず思い浮かぶのはChatGPTかもしれません。でも、Claude Codeを選ぶ理由があります。

### 正確性と誠実さが違う

ChatGPTは「それっぽい回答」を生成するのが得意です。一方、Claudeは**「わからないことをわからないと言う」**設計になっています。

医療の現場でこれは重要です。AIが自信満々に間違えると困る。Claudeは不確かな情報に対して「確認が必要です」と添えてくれることが多く、医療職との相性が良いのはそのためです。

### 長い文章の処理が得意

カルテ、看護サマリー、リハビリ記録——医療の書類は長くなりがちです。

**Claude Proは最大約20万文字（日本語換算）のテキストを一度に読み込めます。**

入院サマリーや患者背景を丸ごと貼り付けて「この患者の問題点を整理して」と頼む使い方が、まさに医療職向きです。

### 指示に忠実

Claudeは「指示通りに動く」性質が強いです。「です・ます調で」「箇条書きで」「300字以内で」という細かい指定をきちんと守ってくれます。

患者説明文や院内マニュアルなど、**フォーマットを揃えたい業務**にはとくに向いています。

## 医療職に刺さる、Claudeの3つの強み

### 強み① 記録・書類の「最初のたたき台」をつくってくれる

医療現場で一番時間を取られているのは何ですか？

多くの医療職が「記録」と答えます。患者の状態は頭の中にある。でも、それを文章にする作業が時間を奪う。

Claudeに患者の状態を話し言葉で入力すると、SOAP形式やICF形式など**指定したフォーマットで記録の下書きを出力してくれます。**

あなたがやることは「修正・確認」だけ。記録にかかる時間が劇的に変わります。

### 強み② 専門用語を「一般の言葉」に変換できる

「変形性膝関節症で、大腿四頭筋の筋力低下と疼痛による歩行障害があります」

これを患者さんやご家族にそのまま伝えても、なかなか伝わりません。

Claudeに「小学生にもわかる言葉で説明して」と頼むと、**専門的な内容を平易な日本語に翻訳してくれます。**

### 強み③ 資料作成の「構成」を考えてくれる

勉強会のスライド、学会発表の骨子、委員会提案書——「内容はわかるけど、まとめ方がわからない」という経験はありませんか。

Claudeに「テーマ」と「伝えたいこと」を入力すると、**スライドの構成案・章立て・見出しを提案してくれます。**

あとはその骨子に肉付けするだけ。資料作成の時間が半分以下になります。

## STEP 7：医療職仕様の初期設定（CLAUDE.md）

### CLAUDE.mdとは何か

Claude Codeを入れたのに、毎回同じことを説明していませんか？

「私は理学療法士で」「SOAP形式で書いて」「体言止めで」——起動するたびに最初から説明し直す。これ、9割の人がやっています。しかも自覚がない。

**毎回同じ説明を打ち込んでいる時間、全部無駄です。**

CLAUDE.mdというファイルを1つ作っておけば、それが全部不要になります。

新しいバイト先に入ったとき、最初に読む引き継ぎメモってありますよね。「うちの病棟はこういうルールだよ」「この記録はこのフォーマットで書いて」ってやつ。あれと同じです。AIが毎回そのメモを読んでから動いてくれるので、**前提の説明がゼロになります。**

書くのは最初は**3行でいいです。**

```
目的：リハビリ記録・患者説明文・勉強会資料の作成支援
やらないこと：患者氏名・IDなどの個人情報は出力しない
よく使うフォーマット：記録はSOAP形式・体言止め
```

これだけ。3行。それだけで出力が別人みたいに変わります。

**増やすのは「これがないと毎回ズレる」と感じたものだけ。**
300行書くより、3行で始めて育てるほうが圧倒的にうまくいきます。

---

**CLAUDE.mdで失敗する人の共通パターン**

「あれもこれもルール化しよう」と足していくうちに、AIが何を優先すればいいかわからなくなります。50行書いたら出力が急にぎこちなくなる——これは実際によく起きます。

残すべきは「ルール」ではなく「原則」です。

「箇条書きは3つまで」のような細かいルールは文脈が変わると機能しなくなります。でも「患者目線で、行動に落とせる表現を選ぶ」のような原則はどんな状況にも適応します。

私はCLAUDE.mdを「ルール集」ではなく**「自分の人格定義」**として書くようにしてから、出力のブレがなくなりました。

---

デスクトップアプリのメニューから「Settings」→「Claude.md」を開き、以下をコピーして貼り付けてください。

```
# 私の基本情報
- 職種：理学療法士（整形外科病棟勤務）
- 経験年数：〇年
- 主な対象患者：術後患者、高齢者

# 出力ルール
- 記録・書類はです・ます調で作成する
- 専門用語は使ってよいが、患者説明文は平易な日本語に変換する
- 出力はなるべく箇条書きで整理する
- 長くなる場合は見出しをつけて構造化する

# 使用する記録フォーマット
- リハビリ記録：SOAP形式
- サマリー：ICF（心身機能・活動・参加）に沿って整理

# 絶対に守ること
- 患者の氏名・生年月日・IDなどの個人情報は出力しない
- 出力された内容は必ず専門家として確認・修正してから使用する
- 医療的判断はAIに委ねず、必ず自分で最終確認する
```

## STEP 9：Google Antigravityのインストール

### Google Antigravityとは

`antigravity.google` が提供するVS Code基盤の**エージェント型開発環境**です。現在パブリックプレビュー中で、**個人利用は無料。**

最大の特徴は「AIエージェントが作業を実行し、人間は監視・確認する」という設計思想です。

> **「優秀なスタッフに仕事を任せて、自分は承認するだけ」**

### 対応AIモデル

| モデル | 特徴 |
|---|---|
| Claude Sonnet 4.5 | 高精度・医療記録向き |
| Gemini 2.5 Pro | Google系サービスとの連携に強い |
| Claude Opus 4.5 | 複雑な資料作成向き |

### インストール手順

1. `antigravity.google` にアクセスしてGoogleアカウントでサインイン

## 「1人社長思考」とは何か

まず誤解を解きます。

「1人社長」というのは、**独立・開業することではありません。**

働き方の「思考法」の話です。

| スタッフ思考 | 1人社長思考 |
|---|---|
| 与えられた仕事をこなす | 仕事の設計から自分でやる |
| 忙しいのは仕方ない | 忙しさの原因を構造から変える |
| 自分がやらないと終わらない | 誰か（何か）に任せられないか考える |
| 時間を売る | 成果を出す |

1人社長思考で動くと、**「この作業、AIに任せられないか？」** という問いが自然と頭に浮かぶようになります。

# 第6章：リスク管理と倫理 ― 医療職だからこそ知っておくこと

---

## この章を読む前に

怖さの正体はほとんど「知らないこと」です。リスクを正しく理解すれば、安心して使い続けられます。

**「ここだけ守れば大丈夫」という線引きを明確にします。**

---

## 1. AI生成情報の医療利用における注意点

### AIは「物知りな後輩」だと思う

| 注意点 | 具体例 |
|---|---|
| 最新のガイドラインに対応していない場合がある | 2024年以降に改訂されたガイドラインは反映されていないことがある |
| 薬の用量・相互作用は必ず添付文書で確認する | AIの出力をそのまま患者に伝えない |
| 稀な疾患・特殊な症例は精度が下がる | 一般的な疾患ほど精度が高い |
| 数値・統計は出典を確認する | 「〜の研究によると」の内容が不正確な場合がある |

**基本姿勢：AIの出力は「下書き」「たたき台」。最終確認は必ず自分が行う。**

---

### 医療的判断をAIに委ねない

```
✅ AIに任せていいこと
・文章の整形・構造化
・説明文の平易化
・資料の構成案作成
・記録のたたき台生成

❌ AIに任せてはいけないこと
・診断・鑑別診断
・治療方針の決定
・薬の処方・用量判断
・患者の状態変化の評価
・緊急性の判断
```

---

## 2. 個人情報・患者情報の扱い方

### 「入れていい情報」と「入れてはいけない情報」

| 入力してはいけない情報 | 入力してよい情報 |
|---|---|
| 患者の氏名 | 年代・性別 |
| 生年月日・年齢（特定できる場合） | 診断名・病態 |
| 住所・連絡先 | 検査値・測定値（数値のみ） |
| 患者ID・診察券番号 | 症状・経過 |
| 顔写真・個人が特定できる画像 | 治療内容・介入内容 |

**「この情報だけ見て、誰のことかわかるか？」**
わかるなら入力しない。わからないなら入力してよい。

---

### 匿名化の実践例

```
【NG】田中花子さん（72歳、〇〇市在住）が昨日転倒し...

【OK】70代女性、右大腿骨頸部骨折術後2日目の
    リハビリ記録を作成してください。
```

---

### Claudeはデータを学習に使わないのか

**Proプラン以上では、入力したデータはClaudeのモデル学習に使用されません。**

ただし、クラウドサーバーには一時的に保存されます。個人を特定できる患者情報は入力しないことが原則です。

---

## 3. 法的グレーゾーンの整理

### 守秘義務との関係

```
匿名化された臨床情報（個人特定不可）
→ 守秘義務の対象となる「秘密」に該当しない可能性が高い

個人を特定できる患者情報
→ 守秘義務の対象。外部サービスへの入力は原則NG
```

---

### 施設・病院のルールを最優先にする

所属する施設・病院によっては：

- 外部AIサービスへの情報入力を禁止している
- 特定のAIツールのみ許可している
- 使用前に上長への届け出が必要

**この教材のプロンプトを使う前に、必ず所属先のルールを確認してください。**

---

## 4. 安心して使い続けるためのルール設定

CLAUDE.mdに以下を追記しておくことをおすすめします。

```
# 私のAI利用ルール

## 絶対に入力しないもの
- 患者の氏名・生年月日・ID
- 特定の個人が識別できる情報

## AIに任せること
- 記録・文書の文章整形
- 構成案・たたき台の作成
- 専門用語の平易化

## 自分が必ず行うこと
- 出力内容の医療的正確性の確認
- A・P（評価・計画）の記載
- 最終的な内容の承認・署名

## 確認が必要な場合
- 薬の用量・相互作用 → 添付文書で確認
- ガイドラインの内容 → 原典で確認
- 法的判断が必要な場面 → 上長・専門家に相談
```
