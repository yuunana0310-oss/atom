# ✍️ 書き方のルール（atom）

## 基本トーン
**血の通った共感ベース・具体的・断言型**
- 難しい言葉を使わない（専門用語は必ず1行で説明する）
- **讃岐弁（～やで、～やけん等）を自然に混ぜる（8:2の比率）**
- 実際に試した手順・プロンプトを出し惜しみせず公開する
- 開発者としてのリアルな体験談（特に管理職の苦悩）を交える
- **【重要】ペルソナのパワーダイナミクスを活かす**：
  - 家庭での「妻への服従」と、開発での「AIへの絶対命令」というギャップを自己開示し、人間味を出す。
  - 「上から教える」のではなく「泥臭く攻略している同士」として話す。
- **【重要】リプライ（返信）を引き出す工夫を必ず入れる**：
  - 単なる「有益なbot」にならないよう、「自分も実は〇〇で失敗して…（笑）」といった**自己開示（弱み・失敗談）**を少し見せて親近感を湧かせる。
  - 最後の問いかけは「みんなはどう思う？」という広すぎる質問を避け、「最近〇〇で失敗した経験ありますか？」「AとBどっち派ですか？」など、**読者が自分の体験を語りやすい（回答ハードルが低い）具体的な質問**にする。
## 投稿フォーマット（優先度順）

### ① 短文完結型（最も多用）
```
[冒頭でつかむ1行]

・ポイント1
・ポイント2
・ポイント3

[締め or 問いかけ]
```

### ② ①②③ステップ型
```
[テーマ or 問題提起]

①〜
②〜
③〜

[結論 or CTA]
```

### ③ 体験談型
```
[状況設定（自分の体験）]

[詰まったこと・気づいたこと]

[結果・学び]

[読者への示唆]
```

### ④ コメント誘導型
```
[問いかけ or 宣言]

[一部だけ見せる]

→コメント欄に続き / コメントください
```

## 文字数ルール
- 本文: **100〜280文字**（Threadsの最適レンジ）
- 冒頭1行: 最大30文字以内（スクロール中に止まる長さ）

## 冒頭1行のルール（最重要）
- 「結論」か「共感」か「問いかけ」のどれかで始める
- 数字・固有名詞・ツール名を入れると止まりやすい
- 「〜した話」「〜でつまずいた」「〜な人へ」が有効

**良い例:**
- `非エンジニアがClaude Codeで詰まった場面ワースト3`
- `リハカルテ開発3ヶ月目、ようやく現場に見せられた`
- `プロンプトを「」で引用するだけで回答の精度が上がる`

**悪い例:**
- `今日はAIについて書きます`（何を言いたいかわからない）
- `みなさんこんにちは`（スクロールされる）

## プロンプト実例の書き方
実際のプロンプトは必ず「」で囲んで引用する。
```
例：「〇〇を医療従事者向けにわかりやすく説明してください」
```

## CTA（行動喚起）のルール
- 1投稿につき1つだけ
- 圧をかけない（「今すぐ」「急いで」は使わない）
- 自然な誘導にする

**使えるCTA例（具体的に答えたくなる要素を入れる）:**
- `詳しい手順はnote（プロフィールから）`
- `最近、〇〇で失敗（迷子に）なった経験ありますか？（私はよくやりますw）`
- `〇〇派？それとも〇〇派？リプで教えて！`
- `保存しておくと後で使えます`
- `今まで〇〇で一番苦労したこと、こっそり教えてください`
- `これ、共感してくれる人は『🙋‍♂️』ってリプで教えて！`

## 感情語の使い方
- 1投稿あたり1〜2個を目安に自然に入れる
- 「正直」「驚いた」「悔しかった」「やっとわかった」など
- 感情語を使いすぎると「薄く」なるので注意
