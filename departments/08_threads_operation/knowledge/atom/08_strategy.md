# 🎯 Threads運用戦略（atom）

## なぜThreadsか

| SNS | フォロワー0からバズれるか | テキストだけでOKか | AI自動化しやすいか |
|---|---|---|---|
| X（Twitter） | ✗ フォロワー必須 | ✓ | △ |
| Instagram | △ | ✗ 画像必須 | ✗ |
| TikTok | △ | ✗ 動画必須 | ✗ |
| **Threads** | **✓ 新規でも拡散** | **✓** | **✓ 最強** |

→ **PT×AI**という希少な組み合わせ × アルゴリズム拡散でフォロワー0から戦える

---

## 運用モード

**atomアカウントは「5%人間モード」で運用する（推奨）**

| | 完全自動（0%人間） | **5%人間モード（採用）** |
|---|---|---|
| 投稿ルート | Writer → キュー → 投稿 | Writer → 下書き → 10分レビュー → 投稿 |
| 作業時間 | ほぼゼロ | 1日10分 |
| 投稿品質 | 7点10本 | 9点5本 |
| 収益感 | 月3〜5万 | 桁が変わる可能性 |

**理由**: atomは複数アカウント並行ではなく、1アカウントを本気で育てる戦略のため。

## 差別化戦略（atomの強み）

他のAIアカウントと違う点を武器にする。

| 差別化ポイント | 説明 |
|---|---|
| **PT × AI** | 医療・福祉 × AIは競合が少ない希少な組み合わせ |
| **リアルな葛藤** | 病院管理職の板挟み、家庭での低姿勢、AIへの絶対命令という「多層的な人間性」 |
| **非エンジニア視点** | エンジニアが語るAIではなく、同じ立場の人が語るAI |
| **讃岐弁の温度感** | 記号化されたAI情報に、地方の温度感を乗せて届ける |
| **倫理性と誠実さ** | 職場のルールを守りながらAIを活用する、プロとしての「正しい」ハック |

## 投稿スケジュール

### 1日の投稿数
- **目標: 3〜5本/日**（5%人間モードでの品質重視）
- 上限: 10本/日（bot判定回避のため厳守）
- 最低間隔: 1時間以上

### 推奨時間帯
| 時間帯 | 理由 |
|---|---|
| 7:00〜9:00 | 通勤・朝の情報収集タイム |
| 12:00〜13:00 | 昼休みのスクロールタイム |
| 20:00〜22:00 | 帰宅後のリラックスタイム |

### テーマのローテーション
同じテーマが連続しないよう、以下のサイクルを意識する：

```
月: リハカルテ開発記録（体験談）
火: Claude Code 実践Tips
水: AIツール比較・レビュー
木: プロンプト実例
金: AI×医療現場
土: 軽め（需要確認型・コメント誘導型）
日: 週まとめ or 振り返り
```

## マネタイズ構造

```
投稿本文（有益コンテンツ・売り込みなし）
  └── コメント欄にASPアフィリエイトリンクを自動配置
        ├── Aiarty（AI画像・動画高画質化ツール）
        └── PLAUD（AI連携ボイスレコーダー）
  └── note記事への誘導（有料記事・プロンプト集）
```

### アフィリエイト配置ルール
- 投稿内容とリンク先の関連性が高いときのみ配置
- 本文には入れない（コメント欄のみ）
- 週に2〜3回を上限（多すぎると信頼が下がる）

### note連携
- Threadsで「概要・フック」を発信 → noteで「詳細・プロンプト実例」へ誘導
- CTA: `詳しい手順はnote（プロフィールから）`

### 短期（1ヶ月）

### 中期（3ヶ月）
- フォロワー: 2,000人
- 平均いいね: 30以上/投稿

### 長期（6ヶ月）
- フォロワー: 5,000人
- note有料記事の月間売上: 3万円以上
- Threads + note の合計収益: 月5万円以上

## 成長ロードマップ

### Phase 1: 基盤づくり（1〜2ヶ月目）
- 毎日3〜5本の一貫した投稿で「このアカウントが何を発信するか」を確立
- リハカルテ開発記録で**独自コンテンツ**を積み上げる
- 過去投稿データを分析してどのパターンが伸びるか把握する

### Phase 2: 拡散狙い（3〜4ヶ月目）
- 伸びているテーマを増やし、弱いテーマを減らす（Analystの分析に従う）
- バズった投稿は**シリーズ化**（「リハカルテ開発日記 vol.〇〇」など）
- コメント欄での会話を積極的に拾い、コミュニティを育てる

### Phase 3: 収益化（5〜6ヶ月目）
- フォロワー2,000人到達でアフィリエイト本格稼働
- note記事への誘導を最適化（CTAのA/Bテスト）
- 実績（フォロワー数・収益額）をコンテンツに組み込み「再現性」を証明する

## 品質管理ルール

### Writerの採点基準（7.0未満は棄却・再生成）
1. フックの強さ（冒頭1行でスクロールが止まるか）
2. 有益性（読んで得した感があるか）
3. 具体性（数字・ツール名・手順があるか）
4. テンポ感（読みやすいリズムか）
5. ペルソナ一致度（atomのターゲットに刺さるか）
6. オリジナリティ（atomにしか書けない内容か）
7. 読みやすさ（改行・句読点が適切か）
8. 感情的共鳴（単なるノウハウ提供にとどまらず、弱み等の自己開示によって親近感や共感を生んでいるか）
9. 行動喚起（読者がリプライしやすい「回答ハードルが低い具体的な問いかけ」やCTAがあるか）
10. Threads適性（100〜280文字に収まっているか）

### 重複防止ルール
- 過去100件との類似度0.85以上 → 自動棄却
- 15種類の投稿パターンを強制ローテーション
- 同テーマの連投は最大2件まで

---

## 緊急停止（KILL_SWITCH）

以下の条件で即停止：
- API認証エラー（凍結リスク）
- 1時間以内に10件超の投稿検知
- 重複投稿が2件以上発生
- ユーザーからの停止命令

**解除はユーザー確認なしに行わない。**
