# 📌 参考にする投稿パターン（atom）

## 参考にすべき投稿スタイル

### パターン1: 短文完結（最多使用）
**特徴**: 3〜5行で有益情報を完結させる。最後に一言コメント。
```
Claude Codeで詰まりやすいポイント3つ

・エラー文をそのまま貼らず「何をしたかった」も伝える
・一度に大量の変更を依頼しない
・「なぜこうなるのか」も聞いて理解する

全部やらかした私が言うから間違いない。
```
**atomへの応用例:**
```
スマホのAIアプリで失敗しやすいこと

・最初から長文を投げる
・出力をそのままコピーして使う
・ツールを変えるたびにゼロから設定する

これ全部やってた私が言うから間違いない。
```

---

### パターン2: リスト型（①②③）
**特徴**: 番号付きリストで整理。最後に問いかけ。
```
非エンジニアがAIでアプリ作るときの手順

①まず「何を解決したいか」を1行で書く
②Claude Codeに丸投げせず段階的に依頼する
③エラーが出たらエラー文＋状況をそのまま貼る

この順番を守るだけで全然違う。
```
**atomへの応用例:**
```
1人社長がAIに任せるべき作業TOP3

①定型メール・返信文の下書き
②議事録・要約・報告書の初稿
③SNS投稿のアイデア出し＋構成

これだけで週4〜5時間は戻ってくる。
```

---

### パターン3: 体験談型（ストーリー）
**特徴**: 起承転結で語る。感情の流れがある。
```
リハカルテを作り始めた話

（起）カルテ記録に毎日30分取られていた。
（承）「AIで何とかならないか」とClaude Codeを触り始めた。
（転）コードが全く読めないのにエラーとの格闘3週間。
（結）ようやく動くものができた。

コードが読めなくても形にはなる。
```
**atomへの応用例:**
```
スマホだけでAI副業を始めた話

（起）PCを開く時間がない毎日だった。
（承）通勤中にChatGPTアプリを触り始めた。
（転）音声入力＋AIで記事の下書きができると気づいた。
（結）月3本の記事をスマホだけで完成させた。

環境がないは言い訳だったと知った。
```

---

### パターン4: 誤解解消型
**特徴**: 「〇〇は嘘」「実は〜」で始める。
```
「コードが書けないとアプリは作れない」は嘘。

Claude Codeは指示を出せれば動く。
大事なのはプログラミングより「何を作りたいか」を言語化する力。

私がその証拠。
```
**atomへの応用例:**
```
「AIは難しい」は半分嘘。

スマホアプリだけで使える機能は十分すぎるほどある。
大事なのはツールより「何に使うか」を決めること。

決めた瞬間から使えるようになる。
```

---

### パターン5: コメント誘導型
**特徴**: 本文で問いかけ、詳細をコメント欄へ。
```
Claude Codeで詰まってる人いる？

どこで止まってるかコメントに書いてみて。
同じ経験してたら解決法教えます。
```
**atomへの応用例:**
```
スマホでAI使い始めたけど活用できてない人いる？

何に困ってるかコメントに書いてみて。
使えてる使い方、シェアします。
```

---

## 投稿を参考にするときの注意
- 「形式」だけを参考にする。内容・言い回しは必ずオリジナルに変える
- atomの専門知識（PT・リハカルテ・AI実践）と組み合わせること
- 体験談は必ず一人称で、実際にあったことを書く（フィクションは書かない）
- 投資・仮想通貨・転職・政治・スポーツには一切触れない
