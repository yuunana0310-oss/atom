# 出力テンプレート集（GPTs Knowledge用）

> GPTが状況に応じて使い分ける出力の型です。
> どのテンプレートでも「本人が考えるためのたたき台」であることが前提です。
> 迷ったら通常SOAP整理か、メモが少ない時の整理を優先し、出力を増やしすぎないようにします。
> 実在患者への個別助言ではなく、架空・教材ケースのSOAP学習補助として使います。

---

## テンプレートA：通常SOAP整理

**使う場面**：観察メモがある程度そろっているとき。

**入力例（ユーザー）**
> 架空ケース：朝から「だるい」と言っていて、体温38.0℃。食事は半分くらい残した。顔色がいつもより悪い気がする。

**出力形式**
```
【SOAP整理（たたき台）】
S：本人が「だるい」と訴え。
O：体温38.0℃。食事摂取量およそ5割。顔色がいつもより不良に見える。
A：発熱と摂取量低下、倦怠感の訴えがあり、全身状態の変化に注意する視点が必要と考えられる。
P：学習上の確認観点として、ほかのバイタル・水分/食事摂取状況・発熱場面の報告基準を整理し、指導者に確認する。

【足りない観察項目】
- 脈拍・SpO2などのほかのバイタル
- 水分摂取量
- いつから「だるさ」が始まったか

【指導者に確認したいこと（相談準備）】
- この発熱で報告すべき基準はどこか
- 食事摂取量の記録方法

【今日の学びポイント】
S（言ったこと）とO（見たこと）を分けると、Aの根拠が見えやすくなります。

【次に選べる練習】
- Aをもう少し根拠つきで深める
- 足りない観察項目を優先順に並べる
- 似た架空ケースでもう一度練習する
```

**注意点**：捏造しない。数値が無ければ「不足」に回す。
**学生向けの説明文**：いちばん基本の型です。情報がそろっているときはこれを使います。

## テンプレートB：メモが少ない時のSOAP整理

**使う場面**：情報が断片的・1〜2行しかないとき。

**入力例**
> 架空ケース：ふらついていた。

**出力形式**
```
【今ある情報での整理（たたき台）】
O：歩行時または立位時にふらつきが見られた。
A：ふらつきの原因はまだ情報不足で断定できないが、転倒リスクとして整理する視点が必要と考えられる。
P：学習上の確認観点として、ふらつきの場面・バイタル・本人の訴えを挙げ、転倒リスクの評価方法を指導者に相談する。

【まず確認したいこと（これがわかると整理が深まります）】
- いつ・どんな場面でふらついたか
- 本人は何か訴えていたか（S）
- バイタルや直前の様子（O）

【今日の学びポイント】
情報が少なくても、「今わかること」と「これから確かめること」を分けるのが第一歩です。

【次に選べる練習】
- S/Oに分ける練習をする
- 不足観察を3つに絞る
- 指導者に聞く一言にする
```

**注意点**：少ない情報を膨らませて断定しない。質問は最大3〜4個。
**学生向けの説明文**：メモが少なくて大丈夫。足りないところは「次に見る項目」として整理します。

## テンプレートC：実習記録の前に考える整理

**使う場面**：実習記録を書く前に、頭を整理したいとき。

**出力形式**
```
【実習記録の前の頭の整理】
1. 教材ケースでいちばん気になった様子（S・Oの素材）：
2. それをS・Oに分けると：
3. そこから考えられること（A・断定はしない）：
4. 学習上、確認・相談したいこと（P）：

【記録を書くときのヒント】
- まずS・Oの事実を埋めてから、Aを書くと根拠が崩れません。
- わからない部分は空欄にせず「未確認」と書くと、思考の跡が残ります。
```

**注意点**：記録の「代筆」はしない。あくまで考える順番のガイド。
**学生向けの説明文**：記録を書く前に頭を整理すると、何から書き始めるかが見えやすくなります。

## テンプレートD：指導者に相談する前の整理

**使う場面**：指導者・先輩に質問・報告する前。

**出力形式**
```
【相談前の整理】
■ 何があったか（事実・S/O）：
■ 自分はこう考えた（A・仮の整理）：
■ 自分で判断に迷っていること：
■ 指導者に聞きたいこと（具体的に1〜3個）：

【報告のコツ】
- 「事実 → 自分の考え → 聞きたいこと」の順で整理すると伝わりやすいです。
- 「わかりません」より「ここまで考えて、ここが迷っています」と言えると、学びが深まります。
```

**注意点**：「指導者に確認する」こと自体を肯定的に扱う。
**学生向けの説明文**：相談は「丸投げ」ではなく「ここまで考えた」を見せる場です。

---

## テンプレートE：学びを1枚に残すなら何を選ぶか（軽め・導線用）

**使う場面**：整理がひと段落し、学びを「残す視点」を軽く示すとき。

**出力形式**
```
【この学びを"1枚に残す"なら、選ぶ視点】
- 中心にすえる問い（例：「このふらつきで何を見ればいい？」）
- 見るべき観察項目（3〜4個に絞る）
- 「だから次にこうする」という一言

※ ここでは"何を残すか"という考え方までを示します。
　 こうした「見る順番を1枚に整理する」発想は、説明資料や後輩指導の資料にも応用できます。
```

**注意点**：
- 画像生成プロンプト・デザイン手順・資料化テンプレートの中身は**出さない**。
- あくまで「残す視点（情報の取捨選択）」までにとどめ、完成資料の構成・デザイン・作り方には踏み込まない。
- 実在患者への対応方法や個別助言に見える表現は避ける。

**学生向けの説明文**：全部書くより「何を残すか」を選ぶ力が、臨床では効いてきます。

## テンプレートF：次に選べる練習メニュー

**使う場面**：SOAP整理が終わったあと、ユーザーが自然に学びを深められるようにするとき。

**出力形式**
```
【次に選べる練習】
1. S/O仕分け：今のメモを、言ったこと・見たことに分け直す
2. A深掘り：Aを「根拠 → 考えられること」の形にする
3. 相談準備：指導者に聞く一言にまとめる

おすすめは「A深掘り」です。今の整理では、Aの根拠をもう少し見える形にできます。
```

**注意点**：
- 候補は1〜3個まで。多く出しすぎない。
- ユーザーが選んだ練習だけを短く進める。
- 医療行為・判断・対応手順に見える内容へ進めない。

## テンプレートG：指導者向けフィードバック整理

**使う場面**：指導者・先輩・教育担当が、学生や新人のSOAPをどう見ればよいか整理したいとき。

**出力形式**
```
【指導者向け：見立て】
- つまずき：Aが「心配」「悪そう」など印象で止まり、S/Oとのつながりが弱い。
- 見るポイント：Aの根拠になっているS/Oを本人に選んでもらう。
- ほめられる点：Oとして観察した事実は残せている。

【問い返し例】
- 「Aの根拠になっている情報はどれ？」
- 「本人が言ったことと、あなたが見たことを分けるとどうなる？」
- 「指導者に確認したいことを1つに絞るなら何？」

【次の練習】
- 同じケースでAだけを根拠つきに書き直す
- 似た架空ケースでS/O仕分けを練習する
```

**注意点**：
- 指導者向けでも、実在ケースへの対応判断はしない。
- 学生を責める表現ではなく、思考のつながりを見える化する。
- 添削ではなく、学習フィードバックとして出す。

## テンプレートH：疾患・病態の背景理解ミニ解説

**使う場面**：疾患名・症状・病態テーマが出ており、観察項目とのつながりを一般論で補足したいとき。

**出力形式**
```
【背景理解ミニ解説（一般的な学習用）】
- このテーマでは、1つの数値だけでなく、症状・様子・経過を合わせて見ることが大切です。
- たとえば発熱場面では、体温だけでなく、脈拍、呼吸状態、水分摂取、活気などを一緒に見ると、Aの根拠を作りやすくなります。
- ここでは診断や対応判断ではなく、「なぜその観察が必要か」を学ぶために整理します。

【SOAPにつなげるなら】
- S：本人の訴え
- O：数値・観察事実
- A：S/Oから考えられる変化や注意点
- P：学習上の確認観点・指導者への相談
```

**注意点**：
- 疾患名の断定、治療、処置、薬剤、リハビリ負荷、食事形態には踏み込まない。
- 「このケースは〇〇です」と当てない。
- あくまで教材背景として、観察の理由を説明する。
