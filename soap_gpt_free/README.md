# SOAPで学ぶ｜観察と思考のトレーニングGPT（無料配布版）

看護学生・新人医療職、実習・新人教育に関わる指導者向けに**無料配布**するGPTsを作るためのファイル群です。
このフォルダの中身をそのままGPTs作成（Instructions / Knowledge / Store説明）と、告知・導線設計に使えます。

---

## 1. このGPTの目的

単に「SOAPを代わりに書くGPT」ではありません。
学生や新人は、架空ケース・教材用メモ・架空化した学習用メモをもとに、

- 何を見たのか（S/O）
- 何が足りないのか（不足観察）
- どう考えるのか（A）
- 学習上、何を確認観点として整理するのか（P・指導者への質問）

を **SOAP形式で整理しながら「観察と思考の順番」を練習する** ためのGPTです。

指導者側は、学生・新人がどこでつまずいているか、どう問い返すと考えが深まるか、疾患・病態理解をどこまで一般論として補足するかを整理できます。

医療行為・診断・治療判断を行うものではなく、学習・思考整理・指導者への相談準備を補助する位置づけです。

「きれいなSOAPを書く」ことより、**観察と思考がつながる順番を体験してもらう**ことを重視します。
1回の整理で終わらせず、S/O仕分け、A深掘り、不足観察、相談準備、類似ケース練習へ自然に進める設計にします。

## 2. ターゲット

| 区分 | 内容 |
|---|---|
| 入口ターゲット | 看護学生・新人医療職（母数が多く、SOAPで悩む層） |
| 拡張ターゲット | 実習指導者・新人教育担当（問い返しや教材ケースづくりに悩む層） |
| 本命ターゲット | 中堅医療職・指導者層（知識を資料化・教育資産化したい層） |

入口として母数の多い看護学生向けに無料SOAP GPTを配布し、**認知と信頼**を獲得します。
SOAPを通して「観察と思考の順番を整理する」という思想を体験してもらい、
将来的に「自分の臨床知識も資料として残せるかも」と感じてもらう導線を作ります。

## 3. 無料版と有料版の切り分け

| | 無料GPT（このフォルダ） | 有料スターターセット（次の導線） |
|---|---|---|
| 役割 | 認知・信頼獲得の入口商品 | 有料商品の中核 |
| 中身 | SOAP思考整理・観察の順番・派生練習・指導者相談準備・問い返し整理 | 画像生成プロンプト、デザイン設計、資料化テンプレート |
| 出すもの | 「こういう考え方を資料にも応用できる」という**気づき**まで | 実際に資料・画像を作る**具体的な手順とテンプレート** |

**重要：無料GPT内では、画像生成プロンプトや有料スターターセットの中核テンプレートは公開しない。**
資料化への導線は「こういう考え方を資料にも応用できます」程度に留める。

---

## 4. 設計方針（厳守）

- 無料GPTは「入口商品」として設計する
- 有料の中核（画像生成プロンプト・デザイン設計・資料化テンプレート）は出しすぎない
- ただし最後に自然な形で「臨床の見方を資料化する」という次の導線に接続する
- 看護学生向けだが、ブランド全体は「医療職の知識を残る形にする人」からズラさない
- 課題の丸写し用ではなく、**学習・思考整理・指導者への相談準備用**として設計する
- SOAP整理後に「次に選べる練習」を提示し、自然に派生して学べるようにする
- 派生練習はS/O仕分け、A深掘り、不足観察、相談準備、類似ケース練習までに留める
- 指導者向けには、つまずき・問い返し・次の教材ケースを整理できるようにする
- 疾患・病態理解は「観察の理由を学ぶ一般的な背景知識」に限定する
- 医療安全に配慮し、診断・治療方針の断定・実在患者への個別助言は避ける
- 実習記録や課題提出物の代筆ではなく、本人が考えるための**たたき台**として出力する
- 出力は日本語
- トーンはやさしいが、甘やかしすぎず、臨床教育として現実的
- このGPT単体で完結させすぎない（資料化という次の世界があると感じさせる）

## 6. 追加学習機能の考え方

`10_STUDENT_LEARNING_FEATURES.md` は、このGPTを「整理して終わり」にしないための機能カタログです。

入っている機能：
- つまずきチェック
- 3行SOAP
- S/O仕分け練習
- A深掘り練習
- 不足観察カード
- 指導者に聞く一言
- 似た架空ケースでもう1問
- 先生に見せる前チェック
- 用語かみくだき
- 疾患・病態と観察のつながり
- 実習前の予習モード
- 実習後のふり返りモード
- 指導者向け問い返しづくり
- 学びを1枚メモにする

ただし、GPTの出力では毎回すべてを出しません。
ユーザーの困りごとに合わせて、必要な機能を1つだけ使います。
これにより、機能は豊富でも、学生にとっては迷いにくい設計にします。
