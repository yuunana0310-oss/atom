## 全ファイル共通で注意すべき点

### 1. 医師指示・施設プロトコル最優先

すべてのKnowledgeファイルで、以下の原則が**最上位**に置かれている：

- 本書の数値・期間・基準は**目安**であり、医師指示に優先しない
- 施設プロトコルがある場合はそれに準じる
- GPTの出力は**参考情報**であり、臨床判断を代替しない

### 2. 断定を避けるクッション語の徹底

各ファイルの患者向け回答では、以下を必ず使う：

- 「目安として」「一般的に」「多くの場合」
- 「個人差があります」
- 「医師の指示があれば」「症状や経過によります」

### 3. エビデンスレベル「★」表記は本書独自分類

本書の各ファイルで使用する「A+／A／B+／B／C」「★」は、本書の独自整理であり、原典のGRADE等のラベルそのものではない。**医学監修者による最終確認**を要監修ポイントに必ず記載している。

### 4. 個別化の促し

各ファイルは「全員共通の最適解」を提示しない。GPT回答時には：

- 術式・経過日数・痛みの状況・医師指示の有無を確認する質問返し
- 患者の社会背景・職業・趣味を踏まえた個別化提案

### 5. 赤旗（危険徴候）の統一運用

各ファイルの「8. 危険徴候・中止基準」はすべて `00_膝関節リハGPT_安全ルール.md` 5-D を**参照する形**で統一されている。施設の連絡フローに合わせて00ファイルを調整すれば、全ファイルに反映される。

### 6. 患者向け／医療者向けの分離

各ファイルの構造：
- 5-G／5-H／5-I 等：**患者向け説明例**（M1モード）
- 5-J／5-I 等：**医療者向け注意点**（M2モード）

GPT実装時は、ユーザーの属性に応じて切り替える（`01_出力モード一覧.md` 参照）。

### 7. 著作権配慮

- 学会CPG・査読論文の文章を**丸写ししない**
- 引用は出典明記の上で要約・自分の言葉で記述
- 図表は出典を明記し、複製ではなく独自整理として作成

### 8. 出典の透明性

| 信頼度 | 例 |
|--------|-----|
| A+ | 学会診療ガイドライン（APTA / 日整会 / OARSI / ACR / Aspetar / EU-US Consensus） |
| A | システマティックレビュー、メタ分析、大規模コホート |
| B+ | 質の高いRCT、実装研究 |
| B | 査読論文、ナラティブレビュー |
| C | 大学病院・専門施設の公開プロトコル（**一例として参照**） |
| D | 個人ブログ・一般Web記事（**原則採用しない**） |

## 運用上の推奨事項

### A. GPT実装時のロード順
1. `00_膝関節リハGPT_安全ルール.md`（必ず最初に読み込む）
2. `01_膝関節リハGPT_出力モード一覧.md`（モード切替のため）
3. ユーザーの質問内容に応じて 02〜10 を参照

### B. 監修プロセス
- 整形外科医（できれば膝関節専門）と理学療法士（経験10年以上推奨）による監修
- **要監修ポイント**（各ファイル9.セクション）を中心に確認
- 監修結果はメモリーまたは別文書に記録し、Knowledgeを更新

### C. 更新頻度
- 学会CPGの改訂があった場合は速やかに反映（年1回チェック）
- 新規エビデンス（メタ分析等）が出た場合は半年〜1年以内に追加検討
- 施設プロトコルが更新された場合は即時反映

### D. 患者向け公開時
- 医療広告ガイドラインに準拠
- 「治る」「効く」などの断定表現は使わない
- 個別の治療効果・経験談は本人同意の上で
- ビフォアフター画像の使用は規制を確認

### E. データ保護
- 患者個人情報を入力された場合の処理ポリシーを明示
- 会話ログの保管期間・暗号化
- アクセス制御
