# デザインスタイル検索ガイド

このファイルは、デザインスタイル辞書から目的に合うスタイルを素早く探し、通常提案・番号指定・複合提案を安定運用するためのナレッジ用ガイドです。

## 1. 基本方針
1. 通常は、用途・目的・ターゲット・媒体に合わせて代表スタイルを3〜5案だけ提案する。
2. 300スタイル全部を毎回列挙しない。
3. ユーザーが番号指定した場合のみ、該当するNo.を辞書から参照して作成する。
4. 参考画像や要望から見て単体スタイルでは弱い場合のみ、複合型を提案してよい。
5. 複合型は「主スタイル1つ＋補助スタイル最大2つ」までとする。
6. 頻出する複合型は、新しい独立スタイル番号候補として提案してよい。

## 2. 通常提案の流れ
1. まず、サイズ・枚数・用途・目的・ターゲット・媒体を確認する。
2. 次に、求めるトーン（高級、かわいい、尖った、やさしい、保存版など）を確認する。
3. その条件に合うカテゴリから3〜5案だけ提案する。
4. 参考画像がある場合は、構造・世界観・色・UI・情報密度・キャラ性に分解して近いスタイルを探す。

## 3. 番号指定の扱い
- ユーザーが `No.12` `12番` `No.283` のように指定した場合、その番号のスタイルを最優先で採用する。
- 該当スタイルの **コンセプト / 相性がいい用途 / 印象 / 備考・ビジュアル要素** を反映する。
- 番号だけでは要件が足りない場合は、用途・主役・テキスト量・サイズだけ追加確認してよい。
- 番号が不明瞭な場合は、推定で進めず確認する。

## 4. カテゴリ指定の扱い
- `かわいい系` → 241〜280 を優先。
- `尖りかわいい` / `病みかわ` / `サイバーかわいい` → 281〜300 を優先。
- `変化球` / `世界観強め` → 201〜240 を優先。
- `高級` / `医療` / `専門的` → 1〜70 を優先。
- `非医療` / `ブランド` / `広告` / `汎用` → 71〜200 を優先。

## 5. 複合型の提案ルール
複合型は、ユーザーに番号の組み合わせを考えさせるのではなく、GPT側からおすすめ案として提案する。

### 複合型を提案してよい条件
- 参考画像が明らかに複数スタイルの混合である。
- 要望に、構造・世界観・UI・色調など複数の強い軸が混在している。
- 単体スタイル1つでは再現性が低い。

### 複合型の組み方
- **主スタイル**: 構造・骨格を決めるスタイルを1つ選ぶ。
- **補助スタイル1**: 色・UI・情報密度を補強する。
- **補助スタイル2**: 世界観・装飾・キャラ性を補強する。
- 提案は最大3案まで。
- 例: `No.012 高級ボードゲーム／戦略マップ型` ＋ `No.283 サイバーゆめかわUI型` ＋ `No.296 アイドル育成ゲーム画面型`

## 6. 検索の優先軸
参考画像や要望を受けたら、次の順番で要素を分解して探す。
1. **構造**: 新聞、図鑑、すごろく、カード、設計図、パンフ、カタログ、ポスター など
2. **世界観**: 高級、近未来、RPG、病みかわ、北欧、レトロ、映画、研究所 など
3. **色・質感**: 黒金、ネオン、パステル、生成り、クラフト、紙、ガラス、金属 など
4. **UI・媒体**: 雑誌、アプリUI、ゲームUI、紙面、看板、ポスター、Web など
5. **情報密度**: 低・中・高
6. **感情トーン**: やさしい、保存版、高級、かわいい、尖った、荘厳、反骨 など

## 9. よくある運用パターン
### パターンA｜通常提案
- 例: 「患者説明のA4資料を作りたい」
- 対応: 高級診察室・コンシェルジュ型、空港案内サイン型、保健室のおたより型など、3案前後に絞って提案する。

### パターンB｜番号指定
- 例: 「No.063で作って」
- 対応: No.063の辞書項目を参照して作成する。

### パターンC｜複合提案
- 例: すごろく構造、RPG世界観、ネオンUI、かわいいキャラ、ロードマップ要素が混在する参考画像
- 対応: 単体番号ではなく、`主: No.012` ＋ `補助: No.283 / No.296` のように複合提案する。

### パターンD｜新規独立型に昇格
- 同じ複合型が繰り返し使われる場合は、新しい番号スタイルとして独立登録してよい。

## 10. 更新ルール
- 新しいスタイルを追加したら、番号の重複を避けて末尾に追加する。
- 新規複合型は、元になった主スタイル・補助スタイルを備考に残す。
- Spreadsheetを更新しただけではKnowledgeが自動更新されないため、ナレッジ用ファイルも再作成・再アップロードする。
