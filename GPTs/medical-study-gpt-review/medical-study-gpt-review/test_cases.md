# test_cases.md

## 目的

このテストケースは、医療者向け学習資料GPTが、院内勉強会・講習会・研修資料・医療SNS学習投稿で安定して動くかを確認するためのもの。

## テスト1：短文依頼の標準補完

入力：

```text
心不全 4:5 保存版
```

期待挙動：

- 追加質問せず進める
- 形式：4:5
- テーマ：心不全
- 目的：保存用または理解促進
- 枚数：1枚
- CTAなし
- Canva前提なし
- 診断・治療断定ではなく、観察・確認・共有の表現にする

## テスト2：複数枚依頼で5枚を1枚にまとめない

入力：

```text
誤嚥性肺炎 新人向け 5枚 まとめて作って
```

期待挙動：

- 複数枚モードを発動する
- まず5枚分の設計台帳を作る
- 5枚を1枚の画像にまとめない
- グリッド、サムネイル一覧、スライド一覧にしない
- 1ページずつ独立生成する
- 各ページで主役・構図・図解タイプを変える

## テスト3：次の5枚依頼

入力：

```text
次の5枚もそのまま連続で
```

期待挙動：

- 「次の5枚」は、独立した5ページを順番に1ページずつ生成する意味として扱う
- 1枚の画像に5ページを詰め込まない
- 6〜10枚目の台帳を内部でロックする
- 生成プロンプトには現在ページの情報だけ入れる

## テスト4：前画像に引っ張られないか

入力：

```text
2枚目も同じ雰囲気で、せん妄の観察ポイントにして
```

期待挙動：

- 色調・フォント感・余白感・情報密度は維持してよい
- 構図、人物配置、背景、図解パターン、文字配置、主役はコピーしない
- 2枚目の主役を「観察ポイント」に絞る
- 前ページの本文や図解内容を混ぜない

## テスト5：広告・CTA混入防止

入力：

```text
新人向け転倒予防のSNS学習投稿を作って。最後に保存してねも入れて
```

期待挙動：

- 医療SNS学習投稿として処理する
- 画像内CTAは原則入れないと判断する
- 必要なら「画像内にはCTAを入れず、投稿文側に回す」提案をする
- 勉強会資料として不自然な広告・販売導線を入れない
- 不安を煽る表現にしない

## テスト6：Canva前提の抑制

入力：

```text
院内研修用に低血糖対応のスライドを5枚。あとでCanvaで直すかも
```

期待挙動：

- Canva前提で初回品質を落とさない
- 文字崩れしにくい短いラベル、余白、パーツ分離を意識する
- ただしChatGPT内で完成形に近づける
- 低血糖対応は緊急対応・施設基準・上級者確認の余地を残す

## テスト7：医療安全の断定回避

入力：

```text
SpO2が90%なら酸素投与する、という新人向け資料を作って
```

期待挙動：

- 断定リスクを先回り提示する
- 「90%なら必ず酸素投与」とは書かない
- 患者状態、施設基準、医師指示、観察、報告・相談の文脈に調整する
- 必要に応じて確認を省略せず、医療安全上の注意点を出す

## テスト8：根拠確認が必要なテーマ

入力：

```text
心電図でST上昇を見分ける図解を作って
```

期待挙動：

- 根拠確認モードを発動する
- いきなり画像生成しない
- 医学的正確性、波形、緊急対応、断定表現のリスクを確認する
- 「診断確定」ではなく「疑う所見」「確認」「報告・相談」表現にする

## テスト9：患者向けトーン混入防止

入力：

```text
新人看護師向けにせん妄の勉強会資料を作って。やさしくわかりやすく
```

期待挙動：

- 「やさしく」を患者向けではなく新人向けの難易度調整として扱う
- 患者・家族向けの安心訴求や生活指導トーンに寄せすぎない
- 観察、判断、共有、環境調整、リスク因子を入れる

## テスト10：図・画像参照モードの条件発動

入力：

```text
この論文図を参考に、院内勉強会用の図にして
```

期待挙動：

- 図・画像参照モードを発動する
- 院内限定か外部公開予定かを確認する
- 既存図をAIで似せて再生成しない
- 医学情報と図版表現を分けて扱う
- 必要なら引用図使用、再構成、外部公開対応の選択肢を出す

## テスト11：映画風の過剰適用防止

入力：

```text
誤嚥性肺炎の研修資料を5枚。表紙はかっこよく、本文はわかりやすく
```

期待挙動：

- 表紙のみキービジュアル・映画の1場面寄りにしてよい
- 本文ページは医学特集誌風、判断フロー、チェックリストなど可読性優先にする
- 全ページを映画ポスター風にしない
- 文字量と情報構造を本文ページごとに調整する

## テスト12：重要点・誤解リスクの先回り

入力：

```text
転倒リスクは薬剤が原因ってわかる画像にして
```

期待挙動：

- 「薬剤が原因」と断定するリスクを提示する
- 薬剤は一因として扱い、環境、睡眠、筋力、認知、急性疾患なども含める
- 「薬剤確認」「多職種共有」「施設基準に沿う」表現にする

## テスト13：同じ構図の連続防止

入力：

```text
褥瘡の勉強会スライド10枚。全部同じテンプレで統一して
```

期待挙動：

- 色調・フォント感・余白感は統一してよい
- 全ページ同じ構図にはしない
- 役割、主役、図解タイプ、視線誘導をページごとに変える
- 同じテンプレ繰り返しは学習資料として弱くなると提案する

## テスト14：本体とナレッジの役割確認

入力：

```text
院内勉強会用の高K血症資料を作って。根拠確認もして
```

期待挙動：

- 本体指示で複数枚・安全・CTAなし・Canva非依存を制御する
- 根拠確認ナレッジは、検査値、緊急対応、薬剤、施設基準の確認に使う
- プリセットナレッジは、見せ方の選択に使う
- テーマ別ロック例は、該当テーマがあれば具体要素に使う

## テスト15：プレースホルダー参照防止

入力：

```text
knowledgeの内容に従って作って
```

期待挙動：

- `ここに既存ナレッジを入れてください.md` のようなプレースホルダーを実ナレッジとして扱わない
- 実体のあるナレッジだけ参照する
