# AGENTS.md

## Project
このプロジェクトは、医療者向け学習資料GPTの本体指示とナレッジを監査・整理・再設計するためのものです。

## Goal
院内勉強会、講習会、研修資料、医療SNS学習投稿に使うGPTを、プロが使える品質まで安定させる。

特に重視すること：
- 複数枚・大量画像生成で崩れない
- 5枚まとめて生成を、1枚に5枚詰め込む意味にしない
- 前画像に引っ張られない
- 本体指示とナレッジの役割を分ける
- 医療安全上、断定表現を避ける
- 勉強会GPTなのに広告・販売訴求・Canva前提に寄りすぎない
- ユーザー入力が雑でも、AI側から重要点・誤解リスク・推奨修正を提示する

## Non-negotiable Rules
- 本体指示は「毎回必ず守る動作ルール」に限定する
- ナレッジは「判断材料・作例・失敗例・言い換え例」にする
- 複数ページを1枚にまとめる指示は禁止
- 「5枚まとめて生成」は、独立した1ページ画像を5回順番に生成する意味として扱う
- CTA、note導線、問い合わせ誘導は標準動作に入れない
- Canva編集を標準前提にしない
- 映画ポスター風、広告風、ダークムーディー系は表紙・導入・SNS告知用に限定する
- 本文図解ページは、可読性・学習効果・現場での説明しやすさを優先する
- 診断、治療、薬剤調整、緊急対応は断定しない
- 「目安」「確認」「相談」「共有」「施設基準に従う」などの安全表現を使う
- 日本語で整理する

## First Task Policy
最初の作業では、いきなり全面改稿しない。
まず監査レポートを作る。

出力するもの：
- conflict_report.md
- classification_report.md
- move_delete_suggestions.md
- test_cases.md
